# 산불 현장 종합 안내 MVP

현장 인력, 상황실, 대피 지원팀이 **한 화면에서 화선/통제선, 대피소, 수자원, 집결지, 후보 동선**을 확인할 수 있도록 만든 시연용 프로토타입이다.

이 보완판은 **현장 입력 포맷 확대**, **수동 차단 구간 반영**, **복수 목표 후보 비교**, **오프라인 PWA 상태 유지**에 초점을 맞췄다.

## 이번 보완에서 추가한 것

- **GeoJSON + KML + GPX 업로드** 지원
- **도로/임도 수동 차단 후보 표시**와 경로 계산 제외
- **목표 유형만 선택했을 때 상위 목표 후보 여러 개 비교**
- 시작점/목표점이 망에서 떨어진 경우 **접속 거리/접속 ETA** 반영
- **유사 경로 분산** 로직으로 대안 경로 다양화
- 경로 카드에 **위험도, 이유, 세부 구간, 즉시 차단 버튼** 추가
- **localStorage 기반 상태 유지**: 시작점, 선택 목표, 차단 구간, 토글 상태
- 레이어 메타에 **업데이트 시각 / 소스 / 지오메트리 유형** 추가

## 경로 계산 방식

도로/임도 GeoJSON을 그래프로 바꾼 뒤, 다음 규칙으로 비용을 계산한다.

1. 기본 이동시간 = 구간길이 / 속도
2. 야간 비권장 구간은 패널티를 부여
3. 산길(trail)은 야간에 추가 패널티
4. 화선 **no-go 버퍼**를 건드리면 해당 구간은 차단
5. 화선 **caution 버퍼**와 겹치는 길이는 비용 가중
6. 통제선 버퍼와 겹치는 구간은 차단
7. 시작점/목표점이 망 밖에 있으면 **접속 거리/시간**을 추가
8. 같은 목표만 반복되지 않도록 **경로 중복 패널티**를 부여
9. 목표가 고정되지 않았으면 **복수 목표 후보를 비교**한다

핵심 파라미터는 `.env.example` 에서 조정할 수 있다.

## 업로드 형식

현재는 **GeoJSON(.geojson/.json)**, **KML(.kml)**, **GPX(.gpx)** 를 지원한다.

### 화선 예시 (GeoJSON)

```json
{
  "type": "FeatureCollection",
  "features": [
    {
      "type": "Feature",
      "properties": { "id": "fireline-current" },
      "geometry": {
        "type": "Polygon",
        "coordinates": [[[128.606,37.446],[128.614,37.446],[128.614,37.454],[128.606,37.454],[128.606,37.446]]]
      }
    }
  ]
}
```

### 통제선 예시 (GPX)

```xml
<gpx version="1.1" creator="demo">
  <trk>
    <name>closure-track</name>
    <trkseg>
      <trkpt lat="37.455" lon="128.609"/>
      <trkpt lat="37.445" lon="128.611"/>
    </trkseg>
  </trk>
</gpx>
```

## UI에서 달라진 점

- 도로/임도 팝업에서 **차단 후보로 표시 / 해제**
- 후보 경로 카드에서 **세부 구간별 차단 버튼**
- 브리핑 카드에서 **현재 선택 경로 핵심 요약**
- 업로드나 차단 변경이 일어나면 **기존 경로를 자동 무효화**
- 새로고침 후에도 **작전 입력 상태를 복원**

## 운영 연계 포인트

실서비스로 키우려면 아래 두 축을 분리하는 게 좋다.

### 1) 작전 레이어
- 화선
- 통제선
- 대피 명령 구역
- 차량 접근 불가 구간
- 현장 제보 수자원
- 수동 차단 임도/도로

### 2) 기반 레이어
- 대피소
- 임시주거시설
- 비상급수/비상급수시설
- 소방용수
- 저수지/관정
- 등산로/임도
- 산사태 위험도 WMS

이 MVP는 `LayerStore`, `importers.py`, `RouteEngine` 에 그 확장 포인트를 넣어 두었다.

## 운영으로 넘길 때 추천 순서

1. 화선 수동 업로드 → 상황실 직접 반영
2. 대피소/수자원 전국 데이터 ETL
3. 임도/도로 네트워크 정합성 검증
4. 차단 구간 입력 UX와 다중 사용자 동기화 강화
5. WMS 중첩(산사태·지형) 활성화
6. 사용자/권한/감사 로그
7. 오프라인 패키징과 현장 리허설

## 주의

샘플 데이터는 **시연용 합성 데이터**다. 실제 작전에는 반드시 공식 데이터와 상황실 판단으로 교차 검증해야 한다.
